## 2.4 Technologies Used

The implementation of the Study Hub platform leverages a modern, scalable technology stack designed to support real-time interaction, efficient document processing, and advanced semantic retrieval. This section outlines the core technologies selected for the production system, combining the prototype's proven backend capabilities with an enhanced frontend architecture.

### 2.4.1 Frontend Frameworks

**React (User Interface):** The final implementation will utilize React, a declarative, component-based JavaScript library for building user interfaces. React's virtual DOM ensures high-performance rendering, which is critical for the chat interface's real-time updates. Its component-based architecture promotes code reusability and modular development, allowing for a distinct separation between the chat visualization, document management, and settings components.

### 2.4.2 Backend & API

**FastAPI (API Server):** The backend is built on FastAPI, a modern, high-performance web framework for building APIs with Python based on standard Python type hints. As demonstrated in the prototype, FastAPI provides automatic API documentation (Swagger UI) and supports asynchronous request handling (`async`/`await`). This asynchronous capability is essential for handling concurrent user requests and long-running LLM inference tasks without blocking the main server thread.

**Python:** Python serves as the primary backend language due to its extensive ecosystem of AI and data processing libraries. It acts as the glue code integrating the vector store, LLM services, and document processors.

### 2.4.3 AI & Machine Learning Integration

**Google Gemini (Generative AI):** The platform utilizes Google's Gemini 2.0 Flash model via the `google-genai` library. Gemini provides the core reasoning and text generation capabilities for the RAG system. Its large context window and multimodal capabilities allow it to effectively synthesize answers based on the retrieved document context.

**SentenceTransformers (Embeddings):** To convert text into vector representations, the system employs the `sentence-transformers` library. These models map sentences and paragraphs to a 384-dimensional dense vector space, capturing semantic meaning rather than just keyword matching. This ensures that user queries retrieve conceptually relevant document chunks even when exact wording differs.

**ChromaDB (Vector Store):** ChromaDB is used as the open-source embedding database. It stores the vector embeddings generated by SentenceTransformers and performs efficient K-Nearest Neighbor (KNN) searches. ChromaDB's ability to persist data locally or run as a server makes it suitable for both the prototype and scalable production environments.

### 2.4.4 Document Processing

**MarkItDown (Content Extraction):** To handle various file formats uploaded by students (PDF, DOCX, etc.), the system uses MarkItDown. This utility converts complex document structures into clean, standardized Markdown text. By normalizing inputs into Markdown, the system ensures that the embedding model receives clean, structured text, improving the quality of downstream retrieval and generation.
