# Product Guidelines: StudySpace

## Design Philosophy
**Minimal & Focused ("Deep Work")**
While the underlying system is complex, the user interface should be clean, spacious, and distraction-free. Avoid "dashboard clutter". The primary view should prioritize the content (PDF or Chat) over navigational chrome.
*   **Principle:** "One task at a time." When reading, hide the chat. When chatting, focus on the conversation.
*   **Layout:** Use collapsible sidebars to maintain focus.

## Visual Identity
**Dark Mode First ("Night Owl")**
Optimized for late-night study sessions. The default theme should use dark greys (`#121212`) rather than pure black to reduce eye strain, with high-contrast text (`#E0E0E0`) for readability.
*   **Accent Color:** Use a soft, encouraging accent color (e.g., Lavender or Teal) for primary actions, avoiding aggressive "alert" colors like pure red unless critical.
*   **Typography:** Clean sans-serif fonts (e.g., Inter or Roboto) with generous line height (1.6) to improve scanability.

## Voice & Tone
**The "Casual Coach"**
The AI persona is a supportive peer tutor, not a strict professor. It should be encouraging, slightly informal, and clear.
*   **Do:** "Great question! Here's how that works..." or "You're on the right track."
*   **Don't:** "The user is incorrect." or overly dense academic jargon without explanation.
*   **Goal:** Reduce anxiety and build confidence.

## Accessibility Standards
**Compliance (WCAG 2.1 AA)**
Ensure the platform is usable by as many students as possible by adhering to standard accessibility guidelines.
*   **Contrast:** Text must meet 4.5:1 contrast ratio.
*   **Navigation:** All interactive elements must be focusable and operable via keyboard (Tab/Enter).
*   **Screen Readers:** Use semantic HTML (`<main>`, `<nav>`, `<button>`) and ARIA labels where necessary.
